Dexter Industries BrickPi Input Port

  1. Documentation
  2. Input and Output Ports
  3. Dexter Industries BrickPi Input Port
This docs page is for kernel version 4.4.44-18-ev3dev-ev3

General Info

device_name brickpi-in-port
port_name prefix in
sysfs class lego-port
# Modes 6

Modes

mode Description
none No sensor
nxt-analog1 NXT/Analog sensor
nxt-color LEGO NXT Color sensor
nxt-i2c2 NXT/I2C sensor
ev3-analog3 EV3/Analog sensor
ev3-uart4 EV3/UART sensor

Notes

  1. The nxt-analog driver will be loaded when this mode is set. You must manually specify the correct driver for your sensor using set_device if you want to use another driver. Any driver with a connection type of NXT/Analog is allowed. ↩

  2. No sensors are loaded by default. You must manually specify the sensor that is connected and its address by using the set_device attribute. This is equivalent to manually loading I2C devices. The sensor port address will be the BrickPi port address with “:i2c” and the decimal I2C address appended. The [BrickPi I2C sensor page] has information regarding the use of sensors connected to a brickpi-in-port. ↩

  3. Only the LEGO EV3 Touch sensor is supported. The driver will load by default. ↩

  4. Only the LEGO EV3 Ultrasonic, Color, Gyro, and Infrared sensors are supported. They cannot be automatically detected, so you must specify the sensor manually using the set_device attribute. No sensors are loaded by default. ↩
